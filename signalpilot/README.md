# SignalPilot Worker — SP-0 (Auth & Audit Spine)

Separate control-plane Worker (`signalpilot`), distinct from the public
`trading-etf` Worker per [ADR-SP-000](../SIGNALPILOT_ROADMAP.md). SP-0 ships
**no trading logic** — only the spine every later mutation must pass through.

## What's here

| File | Role |
| --- | --- |
| `worker.ts` | Entry + router. Public `/health`; authed reads; gated mutations |
| `lib/auth.ts` | Bearer-token verify (constant-time) + replay guard (timestamp window + single-use nonce) |
| `lib/killSwitch.ts` | Server-side `trading_disabled` flag; D1 truth + KV mirror; **fail-closed** |
| `lib/audit.ts` | Append-only, hash-chained `audit_log` writer + chain verifier |
| `lib/http.ts` | json/error helpers, request id, sha256, constant-time compare |
| `env.d.ts` | Merges the `SP_AUTH_TOKEN` secret into the generated `Env` |
| `worker-configuration.d.ts` | Generated by `npm run sp:types` (do not hand-edit) |

## Endpoints (SP-0)

| Method · Path | Auth | Notes |
| --- | --- | --- |
| `GET /health` | none | `{ status, tradingDisabled }` — no secrets |
| `GET /api/control/status` | token | current kill-switch state |
| `GET /api/audit?limit=N` | token | recent rows + chain verification |
| `POST /api/control/kill` | token + replay | disable trading; audited |
| `POST /api/control/resume` | token + replay | enable trading; audited |
| `POST /api/trade/preflight` | token + replay + not-disabled | proves the gate end-to-end; executes nothing |

Mutation requests must send `Authorization: Bearer <SP_AUTH_TOKEN>`,
`X-SP-Timestamp` (unix ms, within ±120s), and a unique `X-SP-Nonce` (8–128 chars).

CSRF is N/A by construction: auth is a non-ambient bearer token (no cookies to
ride). Replay is the residual risk, closed by the timestamp window + nonce.

## One-time provisioning (creates cloud resources — run deliberately)

```bash
NODE=.tools/node-v22.22.3-darwin-arm64/bin/node
WR="$NODE node_modules/wrangler/bin/wrangler.js"
CFG="--config wrangler.signalpilot.toml"

# 1. Create the write DB and control KV, then paste the printed IDs into
#    wrangler.signalpilot.toml (replace the REPLACE_WITH_* placeholders).
$WR d1 create signalpilot-db
$WR kv namespace create SP_CONTROL_KV

# 2. Apply the SP-0 schema (audit_log + control_flags, fail-closed default).
$WR d1 execute signalpilot-db --remote --file=schema/signalpilot-init.sql $CFG

# 3. Set the single-user secret (never commit it).
$WR secret put SP_AUTH_TOKEN $CFG

# 4. Regenerate types (picks up the real binding IDs), typecheck, deploy.
npm run sp:types && npm run sp:deploy
```

## SP-0 exit-gate smoke test (after deploy)

```bash
BASE=https://signalpilot.<your-subdomain>.workers.dev
TOKEN=...   # the SP_AUTH_TOKEN you set

# trading starts DISABLED (fail-closed default)
curl -s $BASE/health                       # tradingDisabled:true

# no token → 401, no state change
curl -s -X POST $BASE/api/control/resume    # 401

# enable, then preflight should pass; disable, then preflight is 423
TS=$(($(date +%s)*1000)); N=$(uuidgen)
curl -s -X POST $BASE/api/control/resume -H "Authorization: Bearer $TOKEN" -H "X-SP-Timestamp: $TS" -H "X-SP-Nonce: $N"
# replay the SAME headers → 409 (nonce reuse)
curl -s -X POST $BASE/api/control/resume -H "Authorization: Bearer $TOKEN" -H "X-SP-Timestamp: $TS" -H "X-SP-Nonce: $N"

# audit trail records every change; chain.ok must be true
curl -s $BASE/api/audit -H "Authorization: Bearer $TOKEN"
```

Gate is met when: unauthenticated mutations are rejected, kill switch toggles
server-side, `/api/trade/preflight` returns 423 while disabled, and every change
appears in `/api/audit` with `chain.ok = true`.

## Day-to-day

```bash
npm run sp:typecheck   # must be zero errors before any deploy
npm run sp:deploy      # typecheck + wrangler deploy (signalpilot config)
```

Never deploy this config to `trading-etf`, and never deploy `trading-etf`'s
`wrangler.toml` here.
